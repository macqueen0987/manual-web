## 필독 (URL·시드 데이터)

| 구분 | 설명 |
|------|------|
| **버전 URL** | 발행 버전 slug는 **8자 해시** (게시 시 버전 이름에서 생성). 예: 이름 `2026.06.03` → URL `/bluefit/f4a15ed7` — **`/bluefit/2026.06.03`은 버전이 아님** |
| **잘못된 2세그먼트 URL** | `/{product}/{segment}`에서 `segment`가 버전 목록에 없으면 **문서 slug**로 해석됨 (기본 발행 버전 + 해당 slug 문서). 혼동 주의 |
| **시드만 있는 제품** | `latest`만 있고 미게시 → 홈 「문서 보기」 후 **빈 트리** 정상. 발행본 있는 제품으로 4장 점검 (예: `uipath-studio` / `2026.05.01`, 또는 Admin에서 게시한 `bluefit` / `f4a15ed7`) |
| **관리자 로그인 중 점검** | 버전 드롭다운에 **초안·latest** 노출됨 → 일반 항목은 **시크릿·로그아웃** 후 재검 |
